# 登录超时提示优化文档

## 问题描述

**现象**：
1. 第一次访问系统时（未登录状态），会弹出两次相同的提示信息
2. 提示信息不友好，显示"请求错误 401"和技术性描述

**问题分析**：

### 原因1：重复的401响应
应用初始化时（`getInitialState`）会同时发起两个请求：
- `fetchUserInfo()` - 获取用户信息
- `fetchMenuInfo()` - 获取菜单信息

未登录时，这两个请求都会返回 401，每个都会触发 `demoResponseInterceptors` 中的 notification

### 原因2：提示信息不友好
原有提示：
```
请求错误 401
您无权访问指定资源，登录状态已过期或未登录，请重新登录！
```

问题：
- 显示错误码对普通用户不友好
- 描述过于技术化
- 使用 error 类型的 notification（红色），给用户压力感

## 解决方案

### 1. 防止重复提示 - 防抖机制

**实现原理**：使用全局标志位 + 定时器实现防抖

```typescript
/** 防止重复显示401提示 */
let hasShown401Notification = false;
let notification401Timer: NodeJS.Timeout | null = null;
```

**工作流程**：
1. 第一个 401 响应到达时，`hasShown401Notification = false`
2. 显示提示后，立即设置 `hasShown401Notification = true`
3. 500ms 内的其他 401 响应会被忽略
4. 500ms 后重置标志位，允许下次显示

**关键代码** (app.tsx:78-102):
```typescript
if (data.code === 401) {
  // 防止重复提示：使用防抖机制，500ms内只显示一次
  if (!hasShown401Notification) {
    hasShown401Notification = true;

    // 根据当前页面状态显示不同的提示
    const isOnLoginPage = window.location.pathname === '/user/login' ||
                         window.location.pathname === '/user/resetPassword';

    if (!isOnLoginPage) {
      // 只在非登录页面显示提示
      notification.warning({
        message: '登录状态已失效',
        description: '您的登录已过期，请重新登录',
        duration: 3,
      });
    }

    // 500ms后重置标志，允许下次显示
    if (notification401Timer) {
      clearTimeout(notification401Timer);
    }
    notification401Timer = setTimeout(() => {
      hasShown401Notification = false;
    }, 500);
  }
  // ... 执行登出和跳转逻辑
}
```

### 2. 优化提示信息

**改进点**：
- ✅ 类型从 `error` (红色) 改为 `warning` (黄色)，减轻用户压力
- ✅ 标题简洁明了："登录状态已失效"
- ✅ 描述通俗易懂："您的登录已过期，请重新登录"
- ✅ 移除技术性错误码显示

**对比**：
| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 类型 | `notification.error` (红色) | `notification.warning` (黄色) |
| 标题 | `请求错误 401` | `登录状态已失效` |

| 显示条件 | 所有401 | 仅非登录页的401 |

### 3. 避免在登录页显示提示

**场景**：用户在登录页输入错误的用户名/密码

**处理**：
- 登录页自己处理登录失败的提示（Login/index.tsx:131-136）
- 响应拦截器检测到在登录页时不显示全局提示

```typescript
const isOnLoginPage = window.location.pathname === '/user/login' ||
                     window.location.pathname === '/user/resetPassword';

if (!isOnLoginPage) {
  // 只在非登录页面显示提示
  notification.warning({...});
}
```

### 4. 优化初始化错误处理

**修改前** (app.tsx:123-130):
```typescript
const fetchUserInfo = async () => {
  try {
    const msg = await getUserInfo();
    return msg.data;
  } catch (error) {
    history.push(loginPath);  // ❌ 直接跳转
  }
  return undefined;
};
```

**问题**：
- `fetchUserInfo` 和 `fetchMenuInfo` 都会在出错时处理跳转
- 导致逻辑混乱，可能重复跳转

**修改后** (app.tsx:149-169):
```typescript
const fetchUserInfo = async () => {
  try {
    const msg = await getUserInfo();
    return msg.data;
  } catch (error) {
    // 不在这里跳转，由拦截器统一处理
    return undefined;
  }
};

const fetchMenuInfo = async () => {
  try {
    const menuData = await fetchMenuData();
    return menuData.data;
  } catch (error) {
    // 菜单获取失败也不处理，由拦截器统一处理
    return undefined;
  }
};
```

**改进**：
- ✅ 统一由响应拦截器处理 401 错误和跳转
- ✅ `fetchUserInfo` 和 `fetchMenuInfo` 只负责获取数据
- ✅ 职责单一，逻辑清晰

### 5. 重置标志位

在页面初始化时重置标志位，确保用户访问时能看到提示：

## 用户体验改进

### 场景1：首次访问（未登录）
**修改前**：
- 显示2次红色错误提示："请求错误 401"
- 用户感到困惑和紧张

**修改后**：
- 显示1次黄色警告提示："登录状态已失效"
- 描述清晰简洁，用户明白需要登录

### 场景2：登录超时后访问
**修改前**：
- 访问任何页面都显示多次错误提示
- 提示信息冗长

**修改后**：
- 仅显示1次友好提示
- 自动跳转到登录页，流程顺畅

### 场景3：在登录页登录失败
**修改前**：
- 可能同时显示登录页的错误和全局提示

**修改后**：
- 仅显示登录页自己的错误提示
- 全局拦截器不干扰登录流程

## 技术要点

### 防抖 vs 节流
本方案使用**防抖（Debounce）**而非节流：
- 节流：固定时间间隔内只执行一次
- 防抖：连续触发时只执行最后一次（或第一次）

选择防抖的原因：
- 401 响应通常集中到达（初始化的2个请求几乎同时返回）
- 需要立即显示第一个提示，忽略后续的
- 500ms 的窗口期足够处理同一批请求

### 时序图

```
用户访问首页（未登录）
    ↓
getInitialState()
    ↓
   并发请求
    ├─→ fetchUserInfo() → 401 → 显示提示（第1次）✓
    └─→ fetchMenuInfo() → 401 → 被防抖忽略 ✗
    ↓
500ms后重置标志
    ↓
用户可以看到下一个401提示
```

## 后续优化建议

1. **Token 刷新机制**
   - 实现 Access Token + Refresh Token 双令牌机制
   - Token 即将过期时自动刷新
   - 减少用户看到 401 的频率

2. **优雅降级**
   - 在 Token 失效前几分钟显示续期提示
   - 用户可选择继续使用或重新登录

3. **记住登录状态**
   - 使用 Refresh Token 实现长期登录
   - 用户下次访问时自动恢复会话

4. **多 Tab 页同步**
   - 使用 `BroadcastChannel` 或 `localStorage` 事件
   - 一个 Tab 登出时，其他 Tab 同步登出
   - 避免多个 Tab 分别显示 401 提示

## 总结

本次优化通过以下手段提升了用户体验：

✅ **防抖机制** - 解决重复提示问题
✅ **友好文案** - 提升可读性和亲和力
✅ **智能判断** - 登录页不显示全局提示
✅ **统一处理** - 集中管理 401 错误逻辑
✅ **视觉优化** - 从红色 error 改为黄色 warning

这些改进让系统的错误提示更加专业、友好、用户体验更佳。
