# 自动登录功能实现文档

## 概述

本文档说明了基于 OAuth2 的自动登录功能的实现细节。该功能使用 refresh_token 机制实现真正的自动登录，而不是简单地记住用户名密码。

## 功能特性

1. **安全的自动登录**: 使用 OAuth2 的 refresh_token 机制，不在本地存储密码
2. **智能 Token 刷新**: 在 token 过期前 5 分钟自动刷新
3. **无感知续期**: 用户操作时自动检查并刷新 token，保持登录状态
4. **401 错误自动恢复**: 当遇到 401 错误时，尝试使用 refresh_token 刷新后重试

## 技术实现

### 1. Token 管理 (`frontend-cloud/src/utils/auth.ts`)

新增了以下功能函数：

```typescript
// Refresh Token 管理
getRefreshToken()       // 获取 refresh_token
setRefreshToken()       // 保存 refresh_token
clearRefreshToken()     // 清除 refresh_token

// 自动登录状态管理
getAutoLogin()          // 获取自动登录状态
setAutoLogin()          // 设置自动登录状态
clearAutoLogin()        // 清除自动登录状态

// Token 过期时间管理
getTokenExpireTime()    // 获取 token 过期时间
setTokenExpireTime()    // 设置 token 过期时间（传入秒数）
clearTokenExpireTime()  // 清除过期时间记录
isTokenExpired()        // 检查 token 是否过期（提前5分钟判定为过期）

// 清除所有认证信息
clearAllAuth()          // 一次性清除所有认证相关数据
```

### 2. Token 刷新 API (`frontend-cloud/src/services/system/login.ts`)

新增了 `refreshToken` 函数：

```typescript
export async function refreshToken(refreshTokenValue: string, options?: { [key: string]: any })
```

该函数向 `/api/oauth2/token` 发送请求，使用 `grant_type=refresh_token` 获取新的 access_token。

### 3. 登录页面更新 (`frontend-cloud/src/pages/user/Login/index.tsx`)

登录成功后，除了保存 access_token，还会：

- 保存 refresh_token
- 保存 token 过期时间
- 根据用户选择保存自动登录状态

### 4. 应用初始化与拦截器 (`frontend-cloud/src/app.tsx`)

#### 4.1 自动登录逻辑

在 `getInitialState` 函数中：

```typescript
// 如果没有 access_token 但有 refresh_token 和自动登录标志
if (!token && refreshTokenValue && autoLogin) {
  // 尝试使用 refresh_token 获取新的 access_token
  const refreshSuccess = await tryRefreshToken();
  if (!refreshSuccess) {
    // 自动登录失败，跳转到登录页
    clearAllAuth();
    history.push(loginPath);
  }
}
```

#### 4.2 请求前拦截器 (authHeaderInterceptor)

在每次请求前：

1. 检查 token 是否即将过期（过期前 5 分钟）
2. 如果过期且开启了自动登录，自动刷新 token
3. 使用防重复刷新机制，避免并发请求时重复刷新

```typescript
if (token && isTokenExpired() && getAutoLogin()) {
  if (!isRefreshing) {
    isRefreshing = true;
    const refreshSuccess = await tryRefreshToken();
    isRefreshing = false;
    // 刷新成功后使用新 token
  } else {
    // 等待正在进行的刷新完成
    token = await new Promise((resolve) => {
      subscribeTokenRefresh((newToken: string) => {
        resolve(newToken);
      });
    });
  }
}
```

#### 4.3 响应拦截器 (demoResponseInterceptors)

当遇到 401 错误时：

1. 检查是否开启了自动登录
2. 尝试使用 refresh_token 刷新 token
3. 刷新成功后，提示用户重新操作
4. 刷新失败，清除所有认证信息并跳转到登录页

## 使用流程

### 用户首次登录

1. 用户在登录页勾选"自动登录"
2. 输入用户名和密码
3. 登录成功后，系统保存：
   - access_token（localStorage）
   - refresh_token（localStorage）
   - autoLogin 标志（localStorage）
   - token 过期时间（localStorage）

### 用户关闭浏览器后再次访问

1. 应用启动，检测到没有 access_token
2. 检查是否有 refresh_token 和 autoLogin 标志
3. 如果有，使用 refresh_token 获取新的 access_token
4. 获取成功，用户无需重新登录即可访问应用
5. 获取失败（refresh_token 过期），跳转到登录页

### Token 过期前自动刷新

1. 用户正常使用应用
2. 在发起请求前，拦截器检查 token 是否即将过期（5 分钟内）
3. 如果即将过期且开启了自动登录，自动使用 refresh_token 刷新
4. 刷新成功，继续原请求
5. 刷新失败，跳转到登录页

### 401 错误自动恢复

1. 请求返回 401 错误
2. 检查是否开启了自动登录
3. 尝试使用 refresh_token 刷新 token
4. 刷新成功，提示用户"Token刷新成功，请重新操作"
5. 刷新失败，显示"登录状态已失效"并跳转到登录页

## 安全考虑

1. **不存储密码**: 只存储 token，不存储用户密码
2. **Refresh Token 保护**: refresh_token 存储在 localStorage，仅在自动登录时使用
3. **提前刷新**: token 过期前 5 分钟就刷新，减少 401 错误
4. **防重复刷新**: 使用标志位和订阅机制，避免并发刷新
5. **清除所有认证**: 退出登录或刷新失败时，清除所有认证信息

## 后端要求

为了支持自动登录功能，后端需要：

1. **支持 OAuth2 refresh_token 授权类型**
   ```
   POST /api/oauth2/token
   Content-Type: application/x-www-form-urlencoded

   client_id=xunlu-admin-web
   &grant_type=refresh_token
   &refresh_token={refresh_token}
   ```

2. **返回格式**
   ```json
   {
     "access_token": "新的访问令牌",
     "refresh_token": "新的刷新令牌（可选）",
     "expires_in": 3600,
     "token_type": "Bearer"
   }
   ```

3. **Refresh Token 配置建议**
   - access_token 有效期: 1-2 小时
   - refresh_token 有效期: 7-30 天
   - 支持 refresh_token 轮换（可选，更安全）

## 测试建议

1. **正常登录测试**: 勾选自动登录，登录后检查 localStorage 中是否保存了所有必要信息
2. **自动登录测试**:
   - 登录后清除 access_token（保留 refresh_token）
   - 刷新页面，检查是否自动登录成功
3. **Token 过期测试**:
   - 修改 token 过期时间为较短时间（如 5 分钟）
   - 等待接近过期，发起请求，检查是否自动刷新
4. **Refresh Token 过期测试**:
   - 使用过期的 refresh_token
   - 检查是否正确跳转到登录页
5. **取消自动登录测试**:
   - 不勾选自动登录
   - 关闭浏览器后再次访问，应该跳转到登录页

## 故障排查

### 自动登录不生效

1. 检查是否勾选了"自动登录"
2. 检查 localStorage 中是否有 `ADMIN_REFRESH_TOKEN` 和 `ADMIN_AUTO_LOGIN`
3. 检查后端是否支持 refresh_token 授权类型
4. 查看浏览器控制台的网络请求，检查 refresh_token 请求是否成功

### Token 刷新失败

1. 检查后端 OAuth2 配置，确保支持 refresh_token
2. 检查 refresh_token 是否过期
3. 检查客户端 ID (`xunlu-admin-web`) 是否正确
4. 查看后端日志，确认刷新请求是否到达并处理

### 频繁跳转到登录页

1. 检查 token 过期时间配置是否合理
2. 检查 refresh_token 是否有效
3. 检查后端是否正确返回 expires_in

## 未来优化方向

1. **Token 刷新队列**: 对于 401 后的 token 刷新，可以实现请求队列，刷新成功后自动重试所有失败的请求
2. **离线检测**: 检测网络状态，离线时不尝试刷新 token
3. **多标签页同步**: 使用 localStorage 事件，在多个标签页之间同步 token 状态
4. **刷新 Token 轮换**: 实现 refresh_token 的轮换机制，提高安全性
5. **Token 即将过期提示**: 在 token 即将过期时（如前 10 分钟）提示用户
